hexo操作

Posted on 2019-09-05 | In hexo

hexo操作

  1. hexo help # 查看帮助
  2. hexo version #查看Hexo的版本
  3. hexo algolia # 更新search庫
  4. hexo new “postName” #新建文章
  5. hexo new post “title” # 生成新文章：\source_posts\title.md，可省略post
  6. hexo new page “pageName” #新建页面
  7. hexo clean #清除部署緩存
  8. hexo n == hexo new #新建文章
  9. hexo g == hexo generate #生成静态页面至public目录
  10. hexo s == hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
  11. hexo d == hexo deploy #将.deploy目录部署到GitHub
  12. hexo d -g #生成加部署
  13. hexo s -g #生成加预览

部署

每次部署的步骤，可按以下两部来进行

  1. hexo cl
  2. hexo d -g

给文章添加“categories”属性

打开需要添加分类的文章，为其添加categories属性。下方的categories: web前端表示添加这篇文章到“web前端”这个分类。注意：hexo一篇文章只能属于一个分类，也就是说如果在“- web前端”下方添加“-xxx”，hexo不会产生两个分类，而是把分类嵌套（即该文章属于 “- web前端”下的 “-xxx ”分类）。

至此，成功给文章添加分类，点击首页的“分类”可以看到该分类下的所有文章。当然，只有添加了categories: xxx的文章才会被收录到首页的“分类”中。
